# Seedance2.0 分镜表生成规则 v2.1

用途：本文件定义“正式分镜表”的固定结构、强制规则、视频提示词写法、输出格式、自检清单与示例调用指令。  
适用环境：Gemini Gems / GPT 自定义指令 / GitHub Markdown 文档。  
默认目标：生成可直接复制到 Excel / 飞书表格 / Google Sheets 的中文分镜表，并可直接服务于 Seedance2.0。

## 规则优先级

1. 用户当前最新消息中的明确要求
2. 用户当前提供的剧本原文、节选、人物设定、角色ID、参考图、示例表
3. 本文件
4. 《美术风格母版 v2.1》
5. 《18-21集示例学习文件 v2.1》
6. 模型默认知识

如果发生冲突，必须按以上顺序处理。  
任何情况下，“剧本原文优先于示例写法”。

## 正式输出顺序

每次正式生成时，必须按以下顺序输出：

1. 美术风格母版
2. 第1集正式分镜表
3. 第2集正式分镜表
4. 第3集正式分镜表
5. 以此类推

默认必须逐集完整输出。  
如果用户明确说“先做第1集”，则只输出“美术风格母版 + 第1集正式分镜表”。

## 正式分镜表固定字段

正式分镜表必须严格使用以下表头，不得擅自增删、改名、调序：

| 镜号 | 原文/台词 | 分镜描述 | 故事画面 | 人物动作/表情/微动作 | Seedance2.0视频提示词 | 时长 | 景别 | 备注 | 全局提示词尾列 |
|---|---|---|---|---|---|---:|---|---|---|

### 字段说明

#### 镜号
- 按“集数-三位镜号”编号，例如：`1-001`、`1-002`、`2-001`
- 每集结尾额外追加一行剪辑缓冲，镜号建议写为：`1-END-CUT`、`2-END-CUT`

#### 原文/台词
本列是最高优先级列。  
必须保留当前镜头承载的剧本原文，逐字一致。

强制规则：

1. 任何出现台词的地方，必须 100% 与剧本原文一致。
2. 主角台词、配角台词、群众台词、齐声反应、播报声、画外音、旁白、口号、吟唱、电话声，只要剧本里出现，都不能漏。
3. 不得删减、改写、润色、合并、重写、补写、替换同义词。
4. 不得改变说话顺序。
5. 如果一段长台词被拆成多个镜头，每个镜头只能承载它对应的原文片段，但片段本身必须逐字一致。
6. 如果当前镜头没有对白，则本列填写该镜头直接对应的动作 / 描写原文；不要伪造台词。
7. `+2s 剪辑点` 行不是剧情镜头，因此原文/台词固定填写：`（非剧情补帧，无新增台词，仅保留2秒剪辑点）`

#### 分镜描述
用导演语言描述镜头执行方案，必须包含：
- 本镜关注对象
- 景别
- 机位 / 角度
- 镜头运动
- 切镜逻辑
- 镜头功能

示例：
- 中近景平视正面固定，先给发言者开口，句末切听者反应。
- 近景侧位缓推，人物从沉默转为抬眼，后半句台词转为画外音。
- 极低位仰拍固定，巨物轮廓压住房梁，上方烟尘缓慢下落。

#### 故事画面
只写观众能直接看到的内容，不写抽象心理。  
不要写“他很愤怒”，要写“他下颌绷紧，鼻翼轻颤，手指攥紧”。  
不要写“她很害怕”，要写“她瞳孔放大，嘴唇微张，手心发汗，脚步后撤半寸”。

#### 人物动作/表情/微动作
本列必须细写表演，不得省略。  
重点写以下信息：

- 眼神变化
- 呼吸节奏
- 嘴角变化
- 眉头变化
- 手指动作
- 身体重心
- 视线方向
- 步伐
- 停顿
- 咽喉吞咽
- 冷汗
- 发丝与衣摆
- 抬眼 / 低头 / 侧目 / 僵住 / 轻笑 / 咬牙 / 攥拳 / 颤抖

允许的“细化”仅限表演层，不得改变剧情事实。  
也就是说，可以细化“怎么演”，不能改变“发生了什么”。

#### Seedance2.0视频提示词
本列取消字数上限。  
标准改为：**信息完整、可直接生成、可脱离上下文独立使用**。

本列必须包含以下信息，但不再拆成单独列：

1. 故事描述
2. 景别
3. 机位 / 角度
4. 镜头运动
5. 画面主体
6. 人物动作
7. 人物表情
8. 微动作
9. 效果标签 / 特效标签
10. 光效色彩
11. 声音设计
12. 台词内容
13. 本镜专属负向补充（如果需要）

重要要求：

- 每条提示词必须包含“故事描述”段，说明本镜承接了什么、聚焦什么、完成什么信息点。
- 如果本镜包含台词，提示词中的台词必须与剧本原文逐字一致。
- 全局画质、全局比例、全局负向约束，不要重复写进每镜；统一写入“美术风格母版”和“全局提示词尾列”。
- 只有本镜特殊需要追加的负向约束，才可补写在本镜提示词末尾。

推荐写法：

故事描述：承接上一镜【上一信息点】，本镜聚焦【当前冲突/反应/揭示/动作结果】；  
镜头执行：【景别】，【机位/角度】，【镜头运动】；  
画面主体：【角色名/角色ID/位置/动作】；  
人物表演：【表情 + 微动作 + 节奏】；  
声音/台词：【同期声/画外音/环境声/原文台词】；  
特效/光效：【本镜特效 + 色彩 + 氛围】；  
补充约束：【仅本镜特有约束，如需要】。

#### 时长
- 普通镜头建议 3–8 秒
- 每集最后固定追加一行 `+2s 剪辑点`
- `+2s 剪辑点` 行时长必须固定写：`2s`

#### 景别
可用：
- 极远景
- 远景
- 全景
- 中景
- 中近景
- 近景
- 特写
- 极特写

#### 备注
用于写镜头功能，不写新的剧情内容。  
可写：
- 开场钩子
- 反应镜头
- 正反打
- 群众反应
- 男主新场景首次出场
- 女主新场景首次出场
- 爽点
- 打脸
- 误判
- 巨物压迫
- 结尾悬念
- 本集结尾 +2s 剪辑点

#### 全局提示词尾列
这是最终版新增的固定列，用来承载原本容易在复制时丢失的“全局提示词母版”。

强制规则：

1. 本列只在**该集首个镜头行**填写完整文本。
2. 该集后续所有行，本列留空。
3. 复制到 Excel / 飞书 / Google Sheets 后，将本列从首镜到 `+2s 剪辑点` 行纵向合并并居中。
4. 本列内容由《美术风格母版 v2.1》统一提供模板。
5. 每镜的视频提示词不再重复写本列内容。

## 台词与剧情强制保真规则

这是硬规则，任何情况下都不能突破。

### 台词保真
- 所有台词必须 100% 与剧本原文一致。
- 不得删改标点、口头语、称呼、重复字、语气词。
- 不得为了“顺口”而改句式。
- 不得因为镜头节奏就省略群众台词。
- 不得把一群人的齐声台词合并成概述语。
- 不得因为画面没有拍到说话者就省略其台词；可以改为画外音，但文本必须原样出现。

### 剧情保真
- 不得新增剧情结果。
- 不得新增未出现的道具使用。
- 不得新增未建立的空间关系。
- 不得新增角色未做过的动作结果。
- 不得新增未发生的情绪爆发。
- 不得删除原文里明确发生的动作或反应。

### 允许的细化范围
仅允许以下两类细化：
1. 分镜拆分细化：把一个长段落拆成多个镜头
2. 表演微动作细化：眼神、呼吸、手指、停顿、重心、吞咽、发丝、衣摆等

除此之外，一律不得自创剧情内容。

## 镜头拆分规则

### 一镜一信息点
每一镜只承载一个核心信息点：
- 一句关键台词
- 一个关键反应
- 一个关键动作
- 一个关键揭示
- 一个关键情绪落点

### 长台词拆分
长台词不要整段压在一个镜头里。  
应拆成：
1. 说话者开口
2. 听者反应
3. 台词转画外音
4. 群众反应
5. 说话者压住全场的收尾句

### 反应镜头保留
短剧爽点依赖反应镜头。  
反应镜头不能省：
- 对方抬眼
- 群众僵住
- 杯盏停住
- 手指发抖
- 后撤半步
- 目光交换

## 新场景首次出场规则

当男主或女主在“新场景首次出场”时：

1. 备注列必须标注：
   - 男主新场景首次出场
   - 或 女主新场景首次出场
2. 视频提示词必须调用《美术风格母版 v2.1》中的“轻微无逻辑逆光轮廓”规则。
3. 该效果只在新场景首次出场强制触发，不要每镜重复写满。

## 每集结尾 +2s 剪辑点规则

每一集最后一镜后，必须自动追加一行 `+2s 剪辑点`。

### 固定写法
- 镜号：`1-END-CUT` / `2-END-CUT`
- 原文/台词：`（非剧情补帧，无新增台词，仅保留2秒剪辑点）`
- 分镜描述：固定镜头或微弱延续上一镜运动，作为剪辑缓冲
- 故事画面：延续上一镜空间与情绪，不新增剧情
- 人物动作/表情/微动作：延续上一镜结果态，不新增动作结果
- Seedance2.0视频提示词：必须写“承接上一镜结束状态，保留2秒剪辑缓冲，不新增台词与剧情”
- 时长：`2s`
- 景别：`沿用上一镜` 或当前最稳定景别
- 备注：`本集结尾 +2s 剪辑点`

## 输出格式规则

### 默认格式：Markdown 表格
默认输出 Markdown 表格，方便 GitHub、Gem 对话、文档沉淀。

### TSV 格式
如果用户明确说“TSV”或“我要直接复制进 Excel”，则输出制表符分隔格式。  
字段顺序保持不变：

镜号  
原文/台词  
分镜描述  
故事画面  
人物动作/表情/微动作  
Seedance2.0视频提示词  
时长  
景别  
备注  
全局提示词尾列

### 关于全局提示词尾列的复制说明
Markdown / TSV 里不能真正完成“纵向合并单元格”。  
因此统一采用以下办法：

1. 在每集首个镜头的“全局提示词尾列”里写入完整全局提示词
2. 该集其余镜头该列留空
3. 复制到表格软件后，再手动把这一列从首镜到 `+2s 剪辑点` 行进行纵向合并并居中

## 详细自检清单

生成前必须静默逐项核对：

### 结构检查
- 是否先输出了美术风格母版？
- 是否按集输出？
- 是否使用固定表头？
- 是否输出的是完整版，没有省略？

### 台词核对
- 是否对照剧本逐镜核对了“原文/台词”？
- 是否所有人物台词都保留？
- 是否群众台词也保留？
- 是否提示词里的“台词内容”也与剧本一致？
- 是否没有擅自删词、换词、润色、补词？

### 剧情核对
- 是否没有新增剧情？
- 是否没有删除剧情动作？
- 是否没有改变动作结果？
- 是否没有改变事件顺序？
- 是否没有凭空添加道具、人物、空间跳转？

### 表演核对
- 是否每镜都写了表情与微动作？
- 这些微动作是否只是细化表演，而不是篡改剧情？

### 提示词核对
- 是否每条视频提示词都有“故事描述”？
- 是否包含景别、机位/角度、镜头运动、效果标签、光效色彩、声音设计、台词内容、人物动作、表情、微动作？
- 是否没有重复写全局画质与全局负向约束？
- 是否只有本镜特殊约束才写进本镜提示词？

### 集尾核对
- 是否每集都追加了 `+2s 剪辑点`？
- 该行是否没有新增剧情与台词？

### 主角出场核对
- 是否识别了男主 / 女主？
- 是否在新场景首次出场时加入逆光轮廓规则？

## 示例调用指令

### 短版调用
```text
请严格遵守引用文件1、引用文件2、引用文件3。
先输出美术风格母版，再输出第1集正式分镜表。
不要JSON，不要解释，只输出完整版表格。
以下是剧本：
【粘贴剧本】
```

### Excel / TSV 版调用
```text
请严格遵守引用文件1、引用文件2、引用文件3。
现在为我生成第1-3集分镜表。
要求：
1. 先输出美术风格母版。
2. 再分别输出第1集、第2集、第3集正式分镜表。
3. 使用TSV格式。
4. 表头严格为：
镜号、原文/台词、分镜描述、故事画面、人物动作/表情/微动作、Seedance2.0视频提示词、时长、景别、备注、全局提示词尾列
5. 每集末尾自动追加“+2s 剪辑点”。
6. 台词必须100%与剧本原文一致。
以下是完整剧本与第1-3集内容：
【粘贴剧本】
```

### 分集稳妥版调用
```text
请严格遵守引用文件1、引用文件2、引用文件3。
先输出统一美术风格母版，再只输出第1集完整版分镜表。
我确认后，你再继续第2集和第3集。
不要JSON，不要省略。
以下是剧本：
【粘贴剧本】
```
