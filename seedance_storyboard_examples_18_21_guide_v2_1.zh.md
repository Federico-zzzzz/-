# 18-21集示例学习文件 v2.1

用途：本文件用于告诉模型“如何学习 18–21 集的既有分镜表写法与颗粒度”。  
注意：本文件内的表格内容是“格式占位示例”，不是项目真实台词。  
如果用户另外上传了真实的 18–21 集示例表（xlsx / tsv / markdown / 贴文），则**真实示例表优先于本文件**。  
本文件只提供学习方向、字段演示与写法标尺，不可把占位台词直接挪用到正式输出中。

## 学习目标

当真实 18–21 集示例表存在时，必须重点学习以下内容：

1. 固定字段顺序
2. 镜头颗粒度
3. 正反打与反应镜头保留方式
4. 群众台词与群体反应的承载方式
5. 视频提示词的组织结构
6. 美术风格母版与全局提示词尾列的关系
7. 每集结尾 `+2s 剪辑点` 的保留方式
8. 不把全局画质与全局负向约束重复写进每镜提示词

## 你应该从真实 18–21 示例里学什么

### 学颗粒度
- 一镜一信息点
- 长台词拆成说话者、听者、群众、压场收尾
- 爽点前后保留反应镜头
- 大动作拆成“意图 → 执行 → 后果”

### 学表格稳定性
- 字段顺序不乱
- 同类镜头写法和命名统一
- 镜号编号规则稳定
- `+2s 剪辑点` 作为编辑行独立存在

### 学提示词层次
- 每镜提示词都含“故事描述”
- 每镜提示词都有动作、表情、微动作、镜头、声音、光效、特效
- 全局质量和全局负向约束不在每镜重复，而是在尾列统一承载

### 学保真原则
- 原文/台词列尽量直接对应剧本
- 不概述、不改写、不偷懒
- 群众台词不丢
- 画外音也不丢

## 你绝对不要从示例里学到的错误

如果真实示例里偶有以下问题，正式生成时不要继承：

1. 漏台词
2. 改写原文
3. 用“众人震惊”替代群众具体反应
4. 每镜都重复写“9:16竖屏、4K、HDR、无字幕……”
5. 不写故事描述，导致单条提示词脱离上下文后不连贯
6. 不保留结尾 `+2s 剪辑点`

## 使用本文件时的硬提醒

1. 本文件的占位表只学习“写法”，不学习“内容”。
2. 正式生成时，原文/台词必须全部替换为用户真实剧本原文。
3. 如果真实 18–21 集示例表与本文件写法冲突，以真实示例表为准。
4. 如果真实示例表存在漏台词、改写台词、无 `+2s 剪辑点` 等问题，正式生成时仍必须遵守引用文件1与引用文件2的硬规则。

## 判断你是否学对了的标准

满足以下条件，才算正确继承了 18–21 集示例风格：

- 字段顺序稳定
- 镜号规则稳定
- 一镜一信息点
- 长台词拆分合理
- 听者反应与群众反应没有被省略
- 视频提示词含“故事描述”
- 全局画质与全局负向约束没有被每镜重复刷屏
- `+2s 剪辑点` 存在
- 男主 / 女主新场景首次出场镜头正确触发逆光轮廓规则
- 所有台词 100% 来自用户真实剧本原文
